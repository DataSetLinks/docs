title: "Query Language Command List"
---

This page provides a reference for constructing queries in the ontology graph engine. Queries allow you to search, filter, and find data stored in interconnected schemas.

## Language reference

#### **`OntologyObject`**

* **Description**: Initializes a query on a specific table/schema.

* **Syntax**: `OntologyObject("TableName")` or `Ontology("TableName")`

* **Example**:
`OntologyObject("Movies")`

#### **`filter`**

 * **Description**: Filters data based on conditions or free-text search.

 * **Syntax**:
* For conditions: `filter("field==value")`

 * **Examples**:
* `OntologyObject("Movies").filter("year==2005")`
* `OntologyObject("Movies").filter("year"<=2005)`

#### **`searchAround`**

* **Description**: Searches objects connected to the current object, optionally specifying the depth of traversal.

* **Syntax**: `searchAround([depth: n])`

* **Example**:
`OntologyObject("Movies").filter("Inception").searchAround(2)`

#### **`find`**

* **Description**: Specifies the table/schema to traverse to. Requires `.searchAround`

* **Syntax**: `find("TableName")`

* **Example**:
`OntologyObject("Actor").searchAround().find("Movies")`

#### **`follow`**

* **Description**: Use only links connected to a specified column

* **Syntax**: `follow("col1","col2")`

* **Example**:
`OntologyObject("Movies").searchAround().find("Actors").follow("director")`

#### **`get`**

* **Description**: Executes the query and retrieves the results. `[optional]`

* **Syntax**: `get()`

* **Example**:
`OntologyObject("Movies").filter(title=="Inception").get()`

## **Error Handling**

If a query returns unexpected results:

1. Check field names and table relationships for correctness.

2. Ensure the depth in `searchAround` is sufficient for your query.

3. Debug by breaking the query into smaller parts and testing them individually.
